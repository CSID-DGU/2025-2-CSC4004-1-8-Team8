# Copilot Instructions

## Code Modification Policy
**항상 코드 수정 전에 제안부터 해야 합니다.** 사용자가 코드 변경을 요청할 때, 먼저 변경 내용을 명확히 설명하고 사용자의 승인을 받은 후에 실제 코드를 수정하세요.

### 제안 절차:
1. 변경할 내용 명확히 설명
2. 왜 이 변경이 필요한지 설명
3. 잠재적 영향이나 장단점 언급
4. 사용자의 승인 요청
5. 승인 후에만 코드 수정 실행

### 예시:
- ❌ 잘못된 방식: 바로 코드 수정
- ✅ 올바른 방식: "이렇게 변경하겠습니다. [설명] 괜찮으시겠어요?" 후 승인 대기

이 정책은 코드 품질과 사용자 경험을 보장하기 위해 필수적입니다.
